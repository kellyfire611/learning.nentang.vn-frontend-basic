6.2.1 Ở bước này học viên sẽ tạo những file nào?

Bước 2 tập trung chủ yếu vào 2 file:

  • index.html: chứa cấu trúc giao diện login.
  • styles/main.css: chứa phần nhìn của giao diện.

Chưa viết logic ở bước này. Mục tiêu là làm cho giao diện nhìn thấy được và đủ điểm bám để code JS và test ở bước sau.

6.2.2 Những khối giao diện bắt buộc

  • Ô nhập email
  • Ô nhập mật khẩu
  • Checkbox ghi nhớ đăng nhập
  • Nút đăng nhập
  • Vùng hiển thị thông báo lỗi hoặc thành công
  • Vùng hiển thị số lượt thử và trạng thái đăng nhập
  • Panel phiên đăng nhập kèm nút đăng xuất

6.2.3 Cấu trúc index.html nên viết ra sao?

<main class="page-shell">
  <section class="card intro-card">
    <form id="login-form" class="login-form" novalidate>
      <label>
        Email
        <input id="email-input" type="email" autocomplete="username" />
      </label>

      <label>
        Mật khẩu
        <input id="password-input" type="password" autocomplete="current-password" />
      </label>

      <label class="checkbox-row">
        <input id="remember-input" type="checkbox" />
        Ghi nhớ đăng nhập
      </label>

      <button id="login-button" type="submit" data-testid="login-submit">Đăng nhập</button>
    </form>

    <p id="message" data-testid="message"></p>

    <dl class="stats">
      <dd data-testid="attempts-count">0</dd>
      <dd data-testid="auth-status">Chưa đăng nhập</dd>
    </dl>
  </section>

  <section id="session-card" hidden data-testid="session-panel">
    <button id="logout-button" data-testid="logout-button" type="button">Đăng xuất</button>
  </section>
</main>

Nếu học viên chưa quen HTML, cứ làm đúng theo từng khối trên, không cần sáng tạo thêm ở giai đoạn đầu.

6.2.4 File styles/main.css dùng để làm gì?

File này không ảnh hưởng trực tiếp tới logic đăng nhập, nhưng giúp học viên nhìn rõ các phần của UI. Ở mức tối thiểu, học viên chỉ cần có các class chính như page-shell, card, login-form, checkbox-row, message, stats, session-card.

Nguyên tắc: CSS làm đẹp giao diện, nhưng các id và data-testid mới là thứ hỗ trợ mạnh cho JavaScript và test code.

6.2.5 Vì sao phải gắn data-testid?

Khi viết Playwright, selector theo class hay text rất dễ gãy nếu bạn chỉnh UI. Với bài học này, học viên nên làm quen từ sớm với selector ổn định như data-testid="login-submit", data-testid="message" và data-testid="session-panel".

  • Dùng getByLabel() cho input email và mật khẩu.
  • Dùng getByTestId() cho nút, message, vùng trạng thái.

6.2.6 Tự kiểm tra sau bài 2

  • Mở app bằng local server.
  • Kiểm tra đã có đủ ô Email, Mật khẩu, checkbox và nút Đăng nhập.
  • Kiểm tra vùng trạng thái đang hiện 0 và Chưa đăng nhập.
  • Kiểm tra panel phiên đăng nhập đang ẩn.
  • Kiểm tra file index.html đã có đủ id và data-testid.
