6.4.1 Bước này cần tạo thư mục và file nào?

Nếu project chưa có thư mục docs/, hãy tạo trước. Trong thư mục này, học viên cần tạo tối thiểu 3 file:

  • docs/test-plan.md
  • docs/test-cases.md
  • docs/student-checklist.md

Đây là các file tài liệu, chưa phải file code test.

6.4.2 test-plan.md nên ghi gì?

Test plan không cần dài, nhưng phải có đủ:

  • Mục tiêu kiểm thử
  • Phạm vi kiểm thử
  • Ngoài phạm vi
  • Môi trường chạy test
  • Tiêu chí pass
# Test Plan - Login App Đơn Giản

## Mục tiêu
- Đảm bảo validate dữ liệu đầu vào hoạt động đúng.
- Đảm bảo luồng đăng nhập và đăng xuất cập nhật state chính xác.

## Phạm vi
- Unit test cho loginStore.
- Business test cho flow sai rồi đúng.
- E2E test cho thao tác trên form login.

6.4.3 test-cases.md nên ghi gì?

File này là nơi liệt kê từng case cụ thể. Mỗi dòng nên có ID, kịch bản, dữ liệu, kỳ vọng.

IDKịch bảnKỳ vọng
TC-01Submit rỗngBáo nhập email, attempts tăng 1
TC-02Email sai định dạngBáo email không hợp lệ
TC-03Thiếu mật khẩuBáo nhập mật khẩu
TC-04Sai tài khoảnBáo sai tài khoản hoặc mật khẩu
TC-05Đăng nhập đúngAuthenticated = true, hiện session panel
TC-06LogoutTrở lại trạng thái chưa đăng nhập

6.4.4 student-checklist.md dùng để làm gì?

Đây là checklist để học viên tự kiểm tra đã làm đủ bài chưa. Ví dụ:

- [ ] Đã viết xong giao diện login
- [ ] Đã có loginStore.js
- [ ] Đã viết xong unit test
- [ ] Đã viết xong business test
- [ ] Đã viết xong E2E test
- [ ] Đã chạy pass npm test
- [ ] Đã chạy pass npm run test:e2e

6.4.5 Vì sao phải viết tài liệu trước?

Nếu không có test plan và test cases, học viên rất dễ viết test theo cảm tính, thiếu negative case và khó biết mình đã hoàn thành bao nhiêu phần của bài.

Luật làm bài: viết tài liệu trước rồi mới sang test code. Đây là thói quen quan trọng nếu sau này làm dự án thật.
