6.5.1 Thứ tự viết unit test

  1. Test normalizeText
  2. Test normalizeEmail
  3. Test isValidEmail
  4. Test login thành công
  5. Test từng case lỗi
  6. Test logout

6.5.2 Ví dụ code

import { describe, expect, test } from "vitest";
import { createLoginStore, normalizeEmail } from "../../src/loginStore.js";

describe("loginStore unit", () => {
  test("normalizeEmail chuyển về chữ thường", () => {
    expect(normalizeEmail(" Student@Example.com ")).toBe("student@example.com");
  });

  test("báo lỗi khi thiếu email", () => {
    const store = createLoginStore();
    const state = store.login("", "123456");
    expect(state.message).toBe("Vui lòng nhập email.");
  });
});

6.5.3 Cách chạy

npm test

Nếu test fail, hãy kiểm tra trước chuỗi message, số attempts và trạng thái authenticated. Đừng nhảy sang E2E khi unit test còn đang đỏ.
