6.5.1 Bước này cần tạo file nào?

Tạo file tests/unit/loginStore.test.js. File này chỉ test logic trong src/loginStore.js, không test trực tiếp HTML.

tests/
└── unit/
    └── loginStore.test.js

6.5.2 Thứ tự viết unit test

  1. Test normalizeText
  2. Test normalizeEmail
  3. Test isValidEmail
  4. Test login thành công
  5. Test từng case lỗi
  6. Test logout

Thứ tự này giúp học viên đi từ dễ đến khó.

6.5.3 Khung code nên viết đầu tiên

import { describe, expect, test } from "vitest";
import {
  createLoginStore,
  isValidEmail,
  normalizeEmail,
  normalizeText
} from "../../src/loginStore.js";

describe("loginStore unit", () => {
  test("normalizeEmail chuyển về chữ thường", () => {
    expect(normalizeEmail(" Student@Example.com ")).toBe("student@example.com");
  });
});

6.5.4 Những test tối thiểu phải có

  • Chuẩn hóa text
  • Chuẩn hóa email
  • Email hợp lệ và không hợp lệ
  • Đăng nhập thành công
  • Thiếu email
  • Email sai định dạng
  • Thiếu mật khẩu
  • Sai tài khoản
  • Logout
test("báo lỗi khi thiếu email", () => {
  const store = createLoginStore();
  const state = store.login("", "123456");

  expect(state.attempts).toBe(1);
  expect(state.authenticated).toBe(false);
  expect(state.message).toBe("Vui lòng nhập email.");
});

6.5.5 Cách chạy và cách đọc lỗi

npm test

Nếu test fail, hãy kiểm tra trước:

  • Chuỗi message expected có khớp 100% với code không
  • attempts có tăng đúng số lần không
  • authenticated đang true hay false
  • user có null hay đã có dữ liệu
Lưu ý: chưa cần nghĩ đến Playwright ở bước này. Chỉ tập trung làm cho logic trong store đúng và test được.
