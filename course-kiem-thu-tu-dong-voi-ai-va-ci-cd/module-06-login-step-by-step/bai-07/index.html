6.7.1 Bước này cần tạo file nào?

Tạo file tests/e2e/login.spec.js. Đây là file test thao tác thật trên trình duyệt.

tests/
└── e2e/
    └── login.spec.js

6.7.2 Những selector nên dùng

  • page.getByLabel("Email")
  • page.getByLabel("Mật khẩu")
  • page.getByTestId("login-submit")
  • page.getByTestId("message")
  • page.getByTestId("session-panel")

Nếu học viên đã làm đúng bài 2 thì các selector này đã có sẵn trong index.html.

6.7.3 Hai case E2E tối thiểu

  1. Submit form rỗng và kiểm tra thông báo lỗi.
  2. Đăng nhập đúng rồi đăng xuất và kiểm tra trạng thái UI ở cả 2 bước.
import { expect, test } from "@playwright/test";

test("đăng nhập thành công rồi đăng xuất", async ({ page }) => {
  await page.goto("/");

  await page.getByLabel("Email").fill("student@example.com");
  await page.getByLabel("Mật khẩu").fill("123456");
  await page.getByTestId("login-submit").click();

  await expect(page.getByTestId("message")).toHaveText("Đăng nhập thành công.");
});

6.7.4 Cấu hình liên quan cần kiểm tra

Trước khi đổ lỗi cho test, hãy nhìn lại 2 chỗ:

  • playwright.config.js: đúng cổng và baseURL chưa
  • package.json: script serve có chạy đúng cổng không
use: {
  baseURL: "http://127.0.0.1:3108",
  headless: true
},
webServer: {
  command: "npm run serve",
  port: 3108
}

6.7.5 Cách debug khi E2E fail

  • Kiểm tra lại baseURL và cổng trong playwright.config.js.
  • Kiểm tra data-testid trong HTML có khớp test không.
  • Kiểm tra text expected có đúng dấu, đúng khoảng trắng.
  • Tự bấm app bằng tay để xác nhận lỗi nằm ở app hay ở test.
Mẹo cho học viên yếu: nếu E2E fail, đừng sửa lung tung cả app lẫn test. Hãy kiểm tra theo đúng thứ tự: app chạy chưa, selector đúng chưa, text đúng chưa.
