6.8.3 Quy trình nộp bài tự kiểm tra

npm test
npm run test:e2e
  1. Chạy lại unit và business test.
  2. Chạy lại E2E test.
  3. Tự bấm app trên trình duyệt ít nhất 3 case.
  4. So lại test case xem đã chuyển đủ sang test code chưa.
  5. Đối chiếu với source mẫu và tự rút ra phần còn yếu.

6.8.4 Học viên nên kiểm tra những lỗi phổ biến nào?

  • Sai tên file hoặc đặt sai thư mục test.
  • Quên type="module" trong package.json.
  • Quên data-testid trong HTML.
  • Message trong code và expected trong test không khớp.
  • Chạy E2E sai cổng vì serve và playwright.config.js không đồng bộ.
Mẹo: khi bài chưa chạy được, đừng sửa nhiều chỗ cùng lúc. Hãy sửa từng lỗi nhỏ rồi chạy lại ngay.
