6.3.1 Tư duy tách logic

Không nhét validate và state trực tiếp vào file HTML. Hãy đặt các hàm như normalizeText, normalizeEmail, isValidEmail, login, logout trong src/loginStore.js.

6.3.2 State tối thiểu

const state = {
  attempts: 0,
  authenticated: false,
  rememberMe: false,
  message: "",
  user: null
};

State này đủ để support cả unit test, business test lẫn E2E test.

6.3.3 Luật nghiệp vụ cần có

  1. Mỗi lần submit đều tăng attempts.
  2. Email rỗng báo Vui lòng nhập email.
  3. Email sai format báo Email không hợp lệ.
  4. Mật khẩu rỗng báo Vui lòng nhập mật khẩu.
  5. Sai tài khoản báo Email hoặc mật khẩu không đúng.
  6. Đúng tài khoản thì set authenticated = true và gán user.
  7. Logout phải trả app về trạng thái chưa đăng nhập.
