9.1 Mocking & Stub

Mock thay thế dependency thật (API call, database, email service) bằng một phiên bản giả để kiểm soát trong test. Stub là mock đơn giản hơn — chỉ trả về giá trị định sẵn. 

// Vitest mock ví dụ
import { vi, describe, it, expect } from 'vitest';
import { placeOrder } from '../../src/js/order.js';
import * as emailService from '../../src/js/emailService.js';

describe('placeOrder', () => {
  it('gửi email xác nhận sau khi đặt hàng', async () => {
    const sendMock = vi.spyOn(emailService, 'sendEmail').mockResolvedValue(true);

    await placeOrder({ items: [], userId: 1 });

    expect(sendMock).toHaveBeenCalledOnce();
    expect(sendMock).toHaveBeenCalledWith(
      expect.objectContaining({ type: 'order_confirmation' })
    );
  });
});

9.2 Test isolation

Mỗi test phải độc lập — không phụ thuộc vào kết quả của test khác.

  • Dùng beforeEach để reset state (giỏ hàng, database giả, mock) trước mỗi test.
  • Không dùng biến global mutable chia sẻ giữa các test.
  • Không có thứ tự chạy test cố định — test nào cũng phải pass khi chạy độc lập.
import { beforeEach, describe, it, expect } from 'vitest';
import { createCart } from '../../src/js/cart.js';

let cart;

beforeEach(() => {
  cart = createCart(); // reset trước mỗi test
});

describe('cart', () => {
  it('bắt đầu rỗng', () => {
    expect(cart.items).toHaveLength(0);
  });
});

9.3 Performance test cơ bản

Kiểm tra thời gian xử lý không vượt ngưỡng chấp nhận được:

it('tính tổng 10,000 sản phẩm dưới 100ms', () => {
  const items = Array.from({ length: 10000 }, (_, i) => ({
    id: i, price: 100, qty: 1,
  }));
  const start = performance.now();
  getTotal(items);
  const elapsed = performance.now() - start;
  expect(elapsed).toBeLessThan(100);
});

9.4 Security test cơ bản

Kiểm tra input validation để tránh lỗ hổng phổ biến:

  • Không cho phép giá trị âm, null hoặc undefined ở các trường quan trọng.
  • Sanitize input tránh XSS khi hiển thị dữ liệu người dùng lên DOM.
  • Kiểm tra hàm không để lộ thông tin nhạy cảm qua error message.
it('ném lỗi khi price là undefined', () => {
  expect(() => addToCart([], { id: 1, price: undefined, qty: 1 }))
    .toThrow();
});

it('sanitize tên sản phẩm — không chứa script tag', () => {
  const name = sanitizeInput('<script>alert(1)</script>');
  expect(name).not.toContain('<script>');
});

9.5 Nguyên tắc code sạch để dễ test

  • Pure functions: Không có side effect, cùng input luôn ra cùng output.
  • Dependency injection: Không khởi tạo dependency bên trong hàm — nhận qua tham số.
  • Single Responsibility: Mỗi hàm làm một việc — nhỏ hơn thì dễ test hơn.
  • Avoid global state: Global mutable state là nguồn gốc của test không ổn định.
