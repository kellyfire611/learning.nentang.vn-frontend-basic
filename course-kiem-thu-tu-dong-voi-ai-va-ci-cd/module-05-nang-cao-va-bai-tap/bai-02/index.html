Bài 2: Bài tập tổng hợp & Ôn tập

Thực chiến 3 cấp độ và câu hỏi ôn tập để kiểm tra lại toàn bộ kiến thức.

Module 5

  • Bài 2: Bài tập tổng hợp & Ôn tập

🟢 Cấp độ 1 — Cơ bản

  1. Bài 1: Tách hàm và viết unit test

    Cho file calculator.js với hàm add(a, b) và divide(a, b) (throw error khi b = 0). Viết đầy đủ unit test bằng Vitest.

  2. Bài 2: Liệt kê test case

    Chức năng: Form đăng ký yêu cầu email hợp lệ, mật khẩu >= 8 ký tự, mật khẩu xác nhận khớp. Liệt kê tất cả test case cần thiết (happy, negative, boundary).

  3. Bài 3: Chạy CI đầu tiên

    Tạo repo GitHub, thêm file .github/workflows/ci.yml chạy npm test mỗi khi push. Xem workflow pass trên tab Actions.

🟡 Cấp độ 2 — Trung cấp

  1. Bài 4: Business test giỏ hàng đầy đủ

    Implement và test đầy đủ module giỏ hàng: thêm/xóa/cập nhật sản phẩm, tính tổng, áp giảm giá, tính phí vận chuyển. Coverage >= 90%.

  2. Bài 5: E2E test trang sản phẩm

    Viết E2E test với Playwright: mở trang, tìm kiếm sản phẩm, thêm vào giỏ, kiểm tra giỏ hàng cập nhật đúng.

  3. Bài 6: Dùng AI sinh test

    Dùng ChatGPT hoặc Copilot sinh unit test cho hàm của bạn. Review kết quả theo checklist (Bài Module 4). Ghi lại điểm AI đúng và điểm AI sai/thiếu.

🔴 Cấp độ 3 — Nâng cao

  1. Bài 7: Full pipeline

    Xây dựng dự án hoàn chỉnh: unit test + business test + E2E test + GitHub Actions CI. Branch protection rule: yêu cầu CI pass trước khi merge PR.

  2. Bài 8: Mocking nâng cao

    Module order.js gọi API gửi email và lưu database. Mock cả 2 dependencies, viết test kiểm tra các trường hợp: gửi thành công, API email lỗi, database lỗi.

  3. Bài 9: Refactor và giữ test xanh

    Nhận code giỏ hàng đã có test đầy đủ. Refactor để cải thiện performance mà không làm bất kỳ test nào fail.

📝 Câu hỏi ôn tập

  1. Test pyramid là gì? Tại sao không nên có quá nhiều E2E test?
  2. Sự khác nhau giữa unit test và business test là gì? Cho ví dụ.
  3. Tại sao cần test isolation? Anti-pattern nào vi phạm isolation?
  4. Mock và stub khác nhau ở điểm gì? Khi nào dùng mock?
  5. CI/CD khác nhau ở điểm gì? Test có vai trò gì trong mỗi giai đoạn?
  6. Kể 3 điều cần kiểm tra khi review test do AI sinh ra.
  7. Shift-left testing nghĩa là gì trong quy trình Agile?
  8. Boundary case là gì? Cho ví dụ với bài toán giảm giá.

📦 Tài nguyên thực hành kèm theo

Đã có sẵn 2 thư mục để học viên thực hành và đối chiếu ngay trong workspace của khóa học.

  • Starter: Mở starter project để học viên tự viết test, tài liệu và CI.
  • Đáp án hoàn chỉnh: Mở source mẫu cuối khóa để đối chiếu sau khi làm xong.
  • Hướng dẫn chi tiết: đọc file README starter và README source mẫu.
