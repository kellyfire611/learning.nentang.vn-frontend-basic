7.1 Prompt sinh test case từ yêu cầu nghiệp vụ

Cung cấp cho AI đủ ngữ cảnh — quy tắc nghiệp vụ cụ thể, không chỉ "viết test cho tôi".

Prompt mẫu:
---
Tôi có yêu cầu nghiệp vụ sau:
- Giỏ hàng tính phí vận chuyển 30,000đ nếu tổng đơn < 500,000đ.
- Miễn phí vận chuyển nếu tổng >= 500,000đ.
- Áp dụng voucher SALE10 giảm thêm 10% trên giá sau phí ship.

Hãy liệt kê các test case cần viết, bao gồm:
happy path, negative case và boundary case.
Định dạng: ID | Tình huống | Input | Kết quả kỳ vọng | Loại
---

7.2 Prompt sinh unit test từ code có sẵn

Prompt mẫu:
---
Đây là hàm JavaScript của tôi:

export function getShippingFee(subtotal) {
  return subtotal >= 500000 ? 0 : 30000;
}

Viết unit test đầy đủ bằng Vitest cho hàm này.
Bao gồm: happy path, boundary case (đúng 500,000) và dưới ngưỡng.
Dùng describe/it/expect. Import từ '../../src/js/pricing.js'.
---

7.3 Checklist review kết quả AI

AI sinh test nhanh nhưng có thể sai hoặc thiếu. Dùng checklist này trước khi commit:

  • ✅ Test có import đúng đường dẫn module?
  • ✅ Tên test mô tả đúng tình huống nghiệp vụ?
  • ✅ Có đủ boundary case ở các ngưỡng điều kiện?
  • ✅ Negative case có kiểm tra đúng loại lỗi ném ra?
  • ✅ Test có bị coupled với implementation (kiểm tra internal state)?
  • ✅ Chạy thử thấy pass — không phải pass vì assert sai?
Bẫy phổ biến: AI thường sinh expect(result).toBeDefined() — điều kiện này luôn pass kể cả khi logic sai. Hãy đổi thành toBe(expectedValue) cụ thể.
