8.1 GitHub Actions là gì?

GitHub Actions là dịch vụ CI/CD tích hợp sẵn trong GitHub. Mỗi khi bạn push code hoặc tạo Pull Request, GitHub tự động chạy workflow bạn định nghĩa trong file YAML ở thư mục .github/workflows/.

Ưu điểm: miễn phí cho repo public, dễ cấu hình, tích hợp sẵn với GitHub ecosystem.

8.2 Workflow mẫu — chạy đầy đủ 3 loại test

# .github/workflows/ci.yml
name: CI — Automated Tests

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  test:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Run unit & business tests
        run: npm test

      - name: Install Playwright browsers
        run: npx playwright install --with-deps chromium

      - name: Run E2E tests
        run: npm run test:e2e

8.3 Tạo file workflow trong dự án

  1. Tạo thư mục .github/workflows/ ở root dự án.
  2. Tạo file ci.yml với nội dung ở trên.
  3. Commit và push lên GitHub.
  4. Vào tab Actions trên GitHub repo để xem workflow chạy.
mkdir -p .github/workflows
# Tạo file ci.yml rồi:
git add .github/workflows/ci.yml
git commit -m "chore: thêm CI workflow chạy automated tests"
git push

8.4 Đọc kết quả CI

  • ✅ Xanh (pass): Tất cả test đều pass — an toàn để merge PR.
  • ❌ Đỏ (fail): Có ít nhất 1 test fail — cần sửa trước khi merge.
  • Nhấn vào job để xem log chi tiết từng bước, tìm test nào fail và vì sao.
Thực hành tốt: Đặt rule "require status checks to pass" trên branch main — không ai có thể merge PR khi CI đang fail.
