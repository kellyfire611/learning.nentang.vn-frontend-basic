2.1 Cấu trúc thư mục chuẩn

Tách code thành 2 phần rõ ràng: src/ chứa logic thuần, tests/ chứa file test.

my-project/
├── src/
│   ├── js/
│   │   ├── cart.js        ← logic nghiệp vụ (dễ import trong test)
│   │   └── utils.js
│   ├── css/
│   │   └── style.css
│   └── index.html
├── tests/
│   ├── unit/
│   │   ├── cart.test.js
│   │   └── utils.test.js
│   ├── business/
│   │   └── discount.test.js
│   └── e2e/
│       └── checkout.spec.js
├── package.json
└── vitest.config.js
Nguyên tắc: Logic nghiệp vụ không được viết trực tiếp trong file HTML (nội dòng vào thẻ <script>). Hãy tách ra thành module JS riêng — chỉ khi đó mới import vào test được.

2.2 Khởi tạo dự án và cài công cụ

  1. Khởi tạo package.json 
    npm init -y
  2. Cài Vitest (unit test & business test) 
    npm install --save-dev vitest
  3. Cài Playwright (E2E test) 
    npm install --save-dev @playwright/test
npx playwright install chromium
  4. Cài http-server (phục vụ file tĩnh cho E2E test) 
    npm install --save-dev http-server

2.3 Cấu hình package.json

{
  "name": "my-project",
  "scripts": {
    "test": "vitest run",
    "test:watch": "vitest",
    "test:e2e": "playwright test",
    "serve": "http-server src -p 3000"
  },
  "devDependencies": {
    "vitest": "^1.0.0",
    "@playwright/test": "^1.40.0",
    "http-server": "^14.0.0"
  }
}

2.4 Cấu hình Vitest

// vitest.config.js
import { defineConfig } from 'vitest/config';

export default defineConfig({
  test: {
    include: ['tests/unit/**/*.test.js', 'tests/business/**/*.test.js'],
    environment: 'node',
  },
});
