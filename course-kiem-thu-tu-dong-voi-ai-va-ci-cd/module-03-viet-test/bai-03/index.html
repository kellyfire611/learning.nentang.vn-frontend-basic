Bài 3: E2E test với Playwright

Mô phỏng thao tác người dùng thật trong trình duyệt, từ click đến kiểm tra kết quả.

6.1 Cấu hình Playwright

// playwright.config.js
import { defineConfig } from '@playwright/test';

export default defineConfig({
  testDir: './tests/e2e',
  use: {
    baseURL: 'http://localhost:3000',
    headless: true,
  },
  webServer: {
    command: 'npm run serve',
    port: 3000,
    reuseExistingServer: !process.env.CI,
  },
});

6.2 Viết E2E test mẫu

// tests/e2e/checkout.spec.js
import { test, expect } from '@playwright/test';

test.describe('Luồng mua hàng cơ bản', () => {
  test('thêm sản phẩm và kiểm tra giỏ hàng', async ({ page }) => {
    await page.goto('/');

    // Nhấn nút "Thêm vào giỏ"
    await page.getByTestId('add-to-cart-1').click();

    // Kiểm tra số lượng trong icon giỏ
    await expect(page.getByTestId('cart-count')).toHaveText('1');
  });

  test('tổng tiền hiển thị đúng', async ({ page }) => {
    await page.goto('/cart.html');
    const total = page.getByTestId('cart-total');
    await expect(total).toBeVisible();
    await expect(total).not.toHaveText('0');
  });
});
Selector tốt nhất: Dùng data-testid attribute trên HTML — không bị ảnh hưởng bởi thay đổi CSS class hay text. Tránh dùng CSS class hay XPath phức tạp.

6.3 Chạy E2E test

# Chạy tất cả E2E test
npm run test:e2e

# Chạy với UI mode (xem trực quan)
npx playwright test --ui

# Chạy một file cụ thể
npx playwright test tests/e2e/checkout.spec.js

# Xem báo cáo HTML sau khi chạy
npx playwright show-report

6.4 Tips viết E2E test ổn định

  • Luôn dùng await expect(...).toBeVisible() thay vì sleep/wait cố định.
  • Tránh test phụ thuộc vào thứ tự — mỗi test phải độc lập.
  • Dùng test.beforeEach để reset trạng thái trước mỗi test.
  • Không hard-code URL — dùng baseURL trong config.
  • Chạy headless trong CI, có thể dùng headed khi debug local.
