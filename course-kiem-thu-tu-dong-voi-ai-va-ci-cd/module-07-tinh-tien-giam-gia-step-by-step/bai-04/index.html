7.4.1 Mục tiêu bài học

Viết tài liệu trước để học viên hiểu “mình test cái gì” trước khi viết code test.

7.4.2 File cần sửa: docs/test-plan.md

Bảng test case bắt buộc có cột phân loại:

  • Happy path: luồng hợp lệ đi đúng.
  • Negative case: dữ liệu sai, thiếu.
  • Boundary case: giá trị biên như 0, 100, 101.
| Mã TC | Cấp độ | Phân loại | Mục tiêu | Dữ liệu vào | Kết quả mong đợi |
|---|---|---|---|---|---|
| Mã TC-UNIT-10 | Unit | Boundary case | Chấp nhận % giảm tại biên | "0", "100" | Không lỗi |

Giải thích: cột phân loại giúp học viên tự kiểm tra độ phủ test, tránh chỉ viết Happy path.

7.4.3 Quy tắc mapping

Mỗi test code đặt tên chứa đúng mã TC. Ví dụ: it("Mã TC-UNIT-10: ...").
