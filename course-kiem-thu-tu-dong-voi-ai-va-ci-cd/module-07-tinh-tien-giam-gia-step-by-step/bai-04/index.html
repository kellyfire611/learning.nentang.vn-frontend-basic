7.4.1 Viết test plan trước khi viết test code

Cập nhật docs/test-plan.md với đầy đủ cột:

  • Mã TC
  • Cấp độ
  • Phân loại: Happy path / Negative case / Boundary case
  • Mục tiêu, dữ liệu vào, kết quả mong đợi

7.4.2 Nguyên tắc mapping

Mỗi test code phải bám đúng một mã TC trong tài liệu để học viên dễ truy vết.
