7.2.1 Mục tiêu bài học

Dựng giao diện form nhập giá và giảm giá để chuẩn bị cho test E2E.

7.2.2 File cần sửa: index.html

<form id="discount-form" novalidate>
  <label>Giá gốc (VNĐ)</label>
  <input type="number" id="price-input" min="0" step="1000" />

  <label>Giảm giá (%)</label>
  <input type="number" id="discount-input" min="0" max="100" step="1" />

  <button type="submit" id="calculate-button">Tính tiền</button>
  <div id="calculation-message" class="calculation-message"></div>
</form>

Giải thích từng id: #price-input và #discount-input để đọc dữ liệu, #calculate-button để submit, #calculation-message để assert ở E2E.

7.2.3 Nối form với hàm xử lý

discountForm.addEventListener("submit", function (event) {
  event.preventDefault();

  const price = document.getElementById("price-input").value;
  const discountPercent = document.getElementById("discount-input").value;
  const message = calculatePayment(price, discountPercent);

  const messageElement = document.getElementById("calculation-message");
  messageElement.textContent = message;

  messageElement.className = "calculation-message";
  if (message.startsWith("Số tiền cần trả:")) {
    messageElement.classList.add("is-success");
  } else {
    messageElement.classList.add("is-error");
  }
});

Giải thích: đoạn này biến kết quả business thành trạng thái UI để user thấy được và E2E test kiểm tra được.

7.2.4 Code đầy đủ file index.html

Học viên có thể dùng đoạn dưới để đối chiếu. Khi mới học, nên gõ lại từng phần thay vì dán toàn bộ để hiểu mỗi dòng làm gì.

<!DOCTYPE html>
<html lang="vi">
<head>
  <meta charset="UTF-8" />
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  <title>Bài thực hành tính tiền sau giảm giá</title>
  <link type="text/css" href="styles/main.css" rel="stylesheet" />
  <script type="module" src="src/discountCalculatorStore.js"></script>
</head>
<body>
  <h1>Tính tiền sau giảm giá</h1>

  <form id="discount-form" method="post" action="#" novalidate>
    <label>Giá gốc (VNĐ)</label>
    <input type="number" id="price-input" min="0" step="1000" />

    <label>Giảm giá (%)</label>
    <input type="number" id="discount-input" min="0" max="100" step="1" />

    <button type="submit" id="calculate-button">Tính tiền</button>
    <div id="calculation-message" class="calculation-message"></div>
  </form>

  <script>
    const discountForm = document.getElementById("discount-form");

    discountForm.addEventListener("submit", function (event) {
      event.preventDefault();

      const price = document.getElementById("price-input").value;
      const discountPercent = document.getElementById("discount-input").value;
      const message = calculatePayment(price, discountPercent);

      const messageElement = document.getElementById("calculation-message");
      messageElement.textContent = message;

      messageElement.className = "calculation-message";
      if (message.startsWith("Số tiền cần trả:")) {
        messageElement.classList.add("is-success");
      } else {
        messageElement.classList.add("is-error");
      }
    });
  </script>
</body>
</html>

7.2.5 Giải thích cho học viên mới

  • type="module" giúp trình duyệt đọc file JavaScript có export.
  • novalidate tắt validate mặc định của trình duyệt để mình tự kiểm tra lỗi bằng JavaScript.
  • id phải viết đúng vì test E2E sẽ tìm đúng các id này.
  • event.preventDefault() giúp form không reload trang sau khi bấm nút.
  • is-success và is-error là dấu hiệu để kiểm tra trạng thái UI.

7.2.6 Lỗi thường gặp

  • Sai id, ví dụ viết price thay vì price-input, làm test không tìm thấy ô nhập.
  • Quên import script discountCalculatorStore.js, làm trình duyệt báo calculatePayment is not defined.
  • Quên preventDefault(), làm trang reload và message biến mất.
