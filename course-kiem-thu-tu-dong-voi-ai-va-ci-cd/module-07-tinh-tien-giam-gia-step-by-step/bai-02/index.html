7.2.1 Mục tiêu bài học

Dựng giao diện form nhập giá và giảm giá để chuẩn bị cho test E2E.

7.2.2 File cần sửa: index.html

<form id="discount-form" novalidate>
  <label>Giá gốc (VNĐ)</label>
  <input type="number" id="price-input" min="0" step="1000" />

  <label>Giảm giá (%)</label>
  <input type="number" id="discount-input" min="0" max="100" step="1" />

  <button type="submit" id="calculate-button">Tính tiền</button>
  <div id="calculation-message" class="calculation-message"></div>
</form>

Giải thích từng id: #price-input và #discount-input để đọc dữ liệu, #calculate-button để submit, #calculation-message để assert ở E2E.

7.2.3 Nối form với hàm xử lý

discountForm.addEventListener("submit", function (event) {
  event.preventDefault();

  const price = document.getElementById("price-input").value;
  const discountPercent = document.getElementById("discount-input").value;
  const message = calculatePayment(price, discountPercent);

  const messageElement = document.getElementById("calculation-message");
  messageElement.textContent = message;

  messageElement.className = "calculation-message";
  if (message.startsWith("Số tiền cần trả:")) {
    messageElement.classList.add("is-success");
  } else {
    messageElement.classList.add("is-error");
  }
});

Giải thích: đoạn này biến kết quả business thành trạng thái UI để user thấy được và E2E test kiểm tra được.
