7.3.1 Mục tiêu bài học

Tách toàn bộ logic sang src/discountCalculatorStore.js để test độc lập, không phụ thuộc UI.

7.3.2 Code mẫu đầy đủ

export function validatePrice(priceInput) {
  if (priceInput === "" || priceInput === null || priceInput === undefined) {
    return "Vui lòng nhập giá gốc!";
  }

  const parsedPrice = Number(priceInput);
  if (!Number.isFinite(parsedPrice) || parsedPrice <= 0) {
    return "Giá gốc phải là số lớn hơn 0!";
  }

  return "";
}

export function validateDiscountPercent(discountPercentInput) {
  if (discountPercentInput === "" || discountPercentInput === null || discountPercentInput === undefined) {
    return "Vui lòng nhập phần trăm giảm giá!";
  }

  const parsedDiscountPercent = Number(discountPercentInput);
  if (!Number.isFinite(parsedDiscountPercent) || parsedDiscountPercent < 0 || parsedDiscountPercent > 100) {
    return "Phần trăm giảm giá phải từ 0 đến 100!";
  }

  return "";
}

export function calculateFinalPrice(price, discountPercent) {
  const discountAmount = (price * discountPercent) / 100;
  return price - discountAmount;
}

export function formatCurrencyVnd(amount) {
  return `${Math.round(amount)} VNĐ`;
}

export function calculatePayment(priceInput, discountPercentInput) {
  const priceError = validatePrice(priceInput);
  if (priceError) return priceError;

  const discountError = validateDiscountPercent(discountPercentInput);
  if (discountError) return discountError;

  const finalPrice = calculateFinalPrice(Number(priceInput), Number(discountPercentInput));
  return `Số tiền cần trả: ${formatCurrencyVnd(finalPrice)}`;
}

7.3.3 Giải thích nhanh theo trách nhiệm

  • validatePrice: chặn rỗng, âm, 0, không phải số.
  • validateDiscountPercent: chặn rỗng và ngoài khoảng [0,100].
  • calculateFinalPrice: chỉ tính toán, không xử lý lỗi.
  • calculatePayment: hàm điều phối luồng nghiệp vụ.

7.3.4 Học viên nên viết theo thứ tự nào?

  1. Viết validatePrice trước vì giá gốc là dữ liệu bắt buộc.
  2. Viết validateDiscountPercent sau vì phần trăm giảm cũng cần kiểm tra riêng.
  3. Viết calculateFinalPrice sau khi dữ liệu đã hợp lệ.
  4. Viết formatCurrencyVnd để chuẩn hóa cách hiển thị tiền.
  5. Cuối cùng viết calculatePayment để nối tất cả hàm nhỏ lại.

7.3.5 Giải thích từng khối code

Khối validate: không tính tiền, chỉ trả về chuỗi lỗi hoặc chuỗi rỗng. Chuỗi rỗng nghĩa là hợp lệ.

Khối tính toán: chỉ nhận số đã hợp lệ. Không nên vừa validate vừa tính trong cùng một hàm vì sẽ khó test.

Khối điều phối: calculatePayment kiểm tra lỗi theo thứ tự. Nếu giá gốc sai thì dừng ngay, không kiểm tra tiếp giảm giá.

7.3.6 Bài tập nhỏ sau khi viết store

Mở DevTools Console của trình duyệt và thử gọi:

calculatePayment("200000", "10")
calculatePayment("", "10")
calculatePayment("200000", "101")

Kết quả mong đợi lần lượt là: tính ra 180000 VNĐ, báo thiếu giá gốc, báo phần trăm giảm không hợp lệ.

7.3.7 Lỗi thường gặp

  • Dùng parseInt làm mất phần thập phân. Trong bài này dùng Number().
  • Quên kiểm tra Number.isFinite, làm chuỗi chữ có thể đi qua sai logic.
  • Trả về số thay vì message đầy đủ trong calculatePayment, làm UI và test không khớp.
  • Quên gắn window.calculatePayment = calculatePayment nếu HTML gọi hàm trực tiếp từ script thường.
