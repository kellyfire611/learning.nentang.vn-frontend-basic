Bài 19: Debug Flexbox bằng DevTools

Bài 19 CSS Flexbox: debug flexbox bằng devtools. Có lý thuyết, ví dụ minh họa, bài tập thực hành và bài giải chi tiết.

Mục tiêu bài học

Dùng DevTools để nhận diện flex container, trục và kích thước item. Trọng tâm áp dụng trong bài này là Debug Flexbox bằng DevTools.

  • Kiểm tra nhanh thuộc tính nào đang bị ghi đè hoặc đặt sai phần tử.
  • Tạo thói quen debug layout thay vì đoán mò.

Lý thuyết cho người mới

DevTools của trình duyệt có overlay cho Flexbox. Bạn có thể bật đường trục, xem khoảng gap, kích thước item và các thuộc tính đang áp dụng.

Khi layout sai, hãy kiểm tra theo thứ tự: container đã display: flex chưa, item có phải con trực tiếp không, thuộc tính đang đặt ở container hay item.

  1. Khoanh vùng phần layout liên quan đến Debug Flexbox bằng DevTools.
  2. Chọn phần tử cha làm flex container và kiểm tra các item con trực tiếp.
  3. Viết CSS tối thiểu trước, sau đó mới thêm màu, padding, border hoặc hiệu ứng.
  4. Thu nhỏ màn hình để kiểm tra item có tràn, chồng chữ hoặc mất khoảng cách không.

Minh họa trực quan

Minh họa bên dưới tập trung vào cơ chế chính của bài Debug Flexbox bằng DevTools. Hãy nhìn hướng di chuyển, khoảng cách và kích thước của từng item trước khi đọc code.

Code tạo minh họa

Đoạn CSS này là phần quan trọng điều khiển cách các item trong khung minh họa được sắp xếp.

.mini-demo {
  display: flex;
  justify-content: center;
  align-items: center;
  gap: 10px;
}

Code ví dụ

Ví dụ này dùng một tình huống gần với giao diện thật để bạn thấy Debug Flexbox bằng DevTools được áp dụng trong thực tế, không chỉ là các ô vuông mẫu.

HTML

<section class="debug-layout">
  <div class="debug-item">Item 1</div>
  <div class="debug-item wide">Item 2 dài hơn</div>
  <div class="debug-item">Item 3</div>
</section>

CSS

.debug-layout {
  display: flex;
  gap: 12px;
  outline: 2px dashed #e85d4f;
}

.debug-item {
  flex: 1;
  outline: 2px solid #0f766e;
}

.debug-item.wide {
  flex: 2;
}

Yêu cầu

Dựng một ví dụ nhỏ cho chủ đề Debug Flexbox bằng DevTools, đặt trong thư mục bai-19-module-01-nhap-mon-flexbox. Bài làm cần có ít nhất một trạng thái desktop và một trạng thái mobile.

  1. Tạo file index.html và style.css.
  2. Viết HTML đúng vai trò nội dung, không đặt class chỉ để trang trí.
  3. Áp dụng đúng thuộc tính Flexbox chính của bài Debug Flexbox bằng DevTools.
  4. So sánh kết quả với phần preview và tự ghi chú điểm khác nhau.
  5. Kiểm tra ở chiều rộng khoảng 360px, 768px và desktop.

Tiêu chí đạt

  • Layout chính dùng Flexbox, không dùng table hoặc float để giả lập.
  • Khoảng cách, căn chỉnh và thứ tự item đúng với mục tiêu bài học.
  • Không có nội dung bị tràn ngang hoặc nút bị chồng lên chữ trên màn hình nhỏ.

Bài giải tham khảo

Bài giải tham khảo ưu tiên CSS ngắn và dễ đọc. Sau khi xem lời giải, hãy tự chỉ ra dòng nào điều khiển container và dòng nào điều khiển item.

<!DOCTYPE html>
<html lang="vi">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Debug Flexbox bằng DevTools</title>
  <style>
    .debug-layout {
      display: flex;
      gap: 12px;
      outline: 2px dashed #e85d4f;
    }
    
    .debug-item {
      flex: 1;
      outline: 2px solid #0f766e;
    }
    
    .debug-item.wide {
      flex: 2;
    }
  </style>
</head>
<body>
  <section class="debug-layout">
    <div class="debug-item">Item 1</div>
    <div class="debug-item wide">Item 2 dài hơn</div>
    <div class="debug-item">Item 3</div>
  </section>
</body>
</html>
