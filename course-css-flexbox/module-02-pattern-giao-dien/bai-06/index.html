Bài 06: Footer nhiều cột bằng Flexbox

Tạo footer nhiều cột tự xuống dòng khi màn hình hẹp.

Mục tiêu Lý thuyết Minh họa Code ví dụ Thực hành Bài giải

Mục tiêu bài học

  • Chia footer thành các cột nội dung có chiều rộng tối thiểu.
  • Dùng flex-wrap để footer không tràn trên mobile.
  • Giữ khoảng cách giữa các nhóm link/form nhất quán bằng gap.

Lý thuyết thực tế

Footer thường có các nhóm nội dung độc lập: thương hiệu, link, hỗ trợ, form nhận tin. Đây là bài toán flex-wrap điển hình.

Mỗi cột nên dùng flex: 1 1 220px: được chia phần trống, được xuống dòng, nhưng không bị ép quá hẹp.

  1. Tạo footer gồm 3 hoặc 4 section.
  2. Bật flex, wrap và gap trên footer.
  3. Đặt từng cột flex: 1 1 220px.
  4. Kiểm tra mobile để mỗi cột vẫn đọc tốt.

Minh họa trực quan

Minh họa này tập trung vào cơ chế chính của bài. Code bên dưới chính là phần điều khiển layout.

NenTang

Học lập trình web qua ví dụ thực tế.

Khóa học

HTML CSS JavaScript

Nhận tin

Code tạo minh họa

.demo-surface {
  display: flex;
  flex-wrap: wrap;
  gap: 12px;
}
.demo-item {
  flex: 1 1 150px;
}

Code ví dụ

Ví dụ này là một tình huống giao diện thật, không phải đoạn CSS rời rạc.

HTML

<footer class="site-footer">
  <section class="footer-col">
    <h3>NenTang</h3>
    <p class="muted">Học lập trình web qua ví dụ thực tế.</p>
  </section>
  <section class="footer-col">
    <h3>Khóa học</h3>
    <a href="#">HTML</a>
    <a href="#">CSS</a>
    <a href="#">JavaScript</a>
  </section>
  <section class="footer-col">
    <h3>Nhận tin</h3>
    <input placeholder="Email của bạn">
    <button>Đăng ký</button>
  </section>
</footer>

CSS

.site-footer {
  display: flex;
  flex-wrap: wrap;
  gap: 20px;
}

.footer-col {
  flex: 1 1 220px;
  display: flex;
  flex-direction: column;
  gap: 8px;
}

Yêu cầu đề bài

Tạo footer gồm thông tin thương hiệu, danh sách link và form nhận tin. Các cột tự xuống dòng trên mobile.

  1. Tạo footer.site-footer chứa ít nhất 3 section.
  2. Đặt footer display: flex, flex-wrap: wrap, gap.
  3. Đặt mỗi section flex: 1 1 220px.
  4. Trong từng section, dùng flex column để link/form xếp dọc.
  5. Kiểm tra mobile không có cột nào bị bó quá hẹp.

Tiêu chí chấm

  • Footer chia cột rõ trên desktop.
  • Các cột xuống dòng tự nhiên trên mobile.
  • Form nhận tin không làm footer tràn ngang.
