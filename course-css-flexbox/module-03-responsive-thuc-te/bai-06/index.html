Bài 06: Đổi bố cục mobile mà không phá thứ tự đọc

Chuyển row sang column an toàn trên mobile, hạn chế lạm dụng order/reverse.

Mục tiêu Lý thuyết Minh họa Code ví dụ Thực hành Bài giải

Mục tiêu bài học

  • Chuyển layout desktop ngang thành mobile dọc bằng media query.
  • Giữ HTML theo thứ tự đọc tự nhiên.
  • Biết khi nào không nên dùng order hoặc reverse.

Lý thuyết thực tế

Main axis là hướng item chạy chính. Với row, item chạy ngang. Với column, item chạy dọc. Cross axis luôn vuông góc với main axis.

Khi đổi flex-direction, ý nghĩa trực quan của justify-content và align-items cũng đổi theo. Đây là nguồn gây nhầm lẫn phổ biến nhất.

  1. Tạo container có 3 item.
  2. Thử row rồi column.
  3. Quan sát item di chuyển theo trục nào khi đổi justify-content.

Minh họa trực quan

Minh họa này tập trung vào cơ chế chính của bài. Code bên dưới chính là phần điều khiển layout.

Yêu cầu đề bài

Tạo layout gồm nội dung chính và sidebar. Desktop nằm ngang, mobile nội dung chính vẫn đọc trước sidebar.

  1. Viết HTML theo thứ tự content trước, sidebar sau.
  2. Desktop dùng flex row.
  3. Mobile chuyển container sang column.
  4. Không dùng column-reverse để sửa thứ tự.
  5. Dùng keyboard Tab để kiểm tra thứ tự focus.

Tiêu chí chấm

  • Desktop có hai cột.
  • Mobile nội dung chính đọc trước sidebar.
  • Không dùng order nếu không có lý do rõ.
