Bài 12: Dự án timeline sự kiện

Bài 12 CSS Flexbox: dự án timeline sự kiện. Có lý thuyết, ví dụ minh họa, bài tập thực hành và bài giải chi tiết.

Mục tiêu bài học

Tạo stepper/timeline có các bước nằm trên một hàng hoặc xuống dòng. Trọng tâm áp dụng trong bài này là Dự án timeline sự kiện.

  • Dùng Flexbox để căn số bước, nhãn và đường nối.
  • Giữ các bước dễ đọc trên mobile.

Lý thuyết cho người mới

Stepper là danh sách các bước theo thứ tự. Container dùng flex để phân phối các bước, mỗi step có thể là flex column để căn số và nhãn.

Khi không đủ chỗ, nên cho step xuống dòng hoặc chuyển thành cột để tránh chữ bị ép quá nhỏ.

  1. Khoanh vùng phần layout liên quan đến Dự án timeline sự kiện.
  2. Chọn phần tử cha làm flex container và kiểm tra các item con trực tiếp.
  3. Viết CSS tối thiểu trước, sau đó mới thêm màu, padding, border hoặc hiệu ứng.
  4. Thu nhỏ màn hình để kiểm tra item có tràn, chồng chữ hoặc mất khoảng cách không.

Minh họa trực quan

Minh họa bên dưới tập trung vào cơ chế chính của bài Dự án timeline sự kiện. Hãy nhìn hướng di chuyển, khoảng cách và kích thước của từng item trước khi đọc code.

Code tạo minh họa

Đoạn CSS này là phần quan trọng điều khiển cách các item trong khung minh họa được sắp xếp.

.mini-demo {
  display: flex;
  justify-content: center;
  align-items: center;
  gap: 10px;
}

Code ví dụ

Ví dụ này dùng một tình huống gần với giao diện thật để bạn thấy Dự án timeline sự kiện được áp dụng trong thực tế, không chỉ là các ô vuông mẫu.

HTML

<ol class="stepper">
  <li><span>1</span>Chọn khóa</li>
  <li><span>2</span>Thực hành</li>
  <li><span>3</span>Nộp bài</li>
</ol>

CSS

.stepper {
  display: flex;
  gap: 12px;
  padding: 0;
  list-style: none;
}

.stepper li {
  flex: 1;
  display: flex;
  align-items: center;
  gap: 8px;
}

Yêu cầu

Tạo danh sách 12 tag kỹ năng. Tag tự xuống dòng, giữ khoảng cách ngang/dọc đều và không tràn màn hình. Chủ đề cần thể hiện rõ trong sản phẩm cuối: Dự án timeline sự kiện.

  1. Tạo container .tags chứa ít nhất 12 span.
  2. Đặt display: flex, flex-wrap: wrap và gap.
  3. Đặt padding cho từng tag, không đặt width cố định quá lớn.
  4. Thử thêm tag có text dài để kiểm tra wrap.
  5. Kiểm tra ở mobile xem có thanh cuộn ngang không.

Tiêu chí đạt

  • Layout chính dùng Flexbox, không dùng table hoặc float để giả lập.
  • Khoảng cách, căn chỉnh và thứ tự item đúng với mục tiêu bài học.
  • Không có nội dung bị tràn ngang hoặc nút bị chồng lên chữ trên màn hình nhỏ.

Bài giải tham khảo

Bài giải tham khảo ưu tiên CSS ngắn và dễ đọc. Sau khi xem lời giải, hãy tự chỉ ra dòng nào điều khiển container và dòng nào điều khiển item.

<!DOCTYPE html>
<html lang="vi">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Dự án timeline sự kiện</title>
  <style>
    .stepper {
      display: flex;
      gap: 12px;
      padding: 0;
      list-style: none;
    }
    
    .stepper li {
      flex: 1;
      display: flex;
      align-items: center;
      gap: 8px;
    }
  </style>
</head>
<body>
  <ol class="stepper">
    <li><span>1</span>Chọn khóa</li>
    <li><span>2</span>Thực hành</li>
    <li><span>3</span>Nộp bài</li>
  </ol>
</body>
</html>

Lỗi thường gặp

  • Quên flex-wrap: wrap làm toàn bộ tag bị ép trên một dòng.
  • Đặt item width quá lớn khiến mobile bị tràn.
  • Dùng margin cho từng tag rồi khó kiểm soát khoảng dọc.
