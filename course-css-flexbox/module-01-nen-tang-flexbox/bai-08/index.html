Bài 08: order, auto margin, nested flex và debug thực tế

Dùng các kỹ thuật thường gặp sau khi đã nắm nền tảng Flexbox.

Mục tiêu Lý thuyết Minh họa Code ví dụ Thực hành Bài giải

Mục tiêu bài học

  • Dùng auto margin để đẩy item trong flex layout.
  • Tạo nested flex khi một item cần căn chỉnh con bên trong.
  • Dùng order có kiểm soát và biết rủi ro accessibility.

Lý thuyết thực tế

Auto margin hấp thụ khoảng trống trên trục tương ứng. margin-left: auto thường đẩy item sang cuối hàng; margin-top: auto đẩy nút xuống cuối card dạng column.

Một flex item có thể là flex container khác. Đây là cách làm card có header/body/footer, toolbar có nhiều nhóm nút, dashboard có nhiều vùng.

order chỉ nên dùng khi thật sự cần đổi hiển thị. Thứ tự DOM vẫn quan trọng với keyboard và screen reader.

  1. Dùng auto margin trước khi nghĩ đến position absolute.
  2. Nếu cần căn nội dung bên trong item, bật flex trên item đó.
  3. Khi dùng order, kiểm tra thứ tự đọc và thứ tự tab.

Minh họa trực quan

Minh họa này tập trung vào cơ chế chính của bài. Code bên dưới chính là phần điều khiển layout.

Bài tập Flexbox

Cần làm

Đọc lý thuyết, tự code lại ví dụ và so sánh với bài giải.

Code tạo minh họa

.demo-surface {
  display: flex;
  gap: 12px;
}
.demo-item:nth-child(4) {
  margin-left: auto;
}

Code ví dụ

Ví dụ này là một tình huống giao diện thật, không phải đoạn CSS rời rạc.

HTML

<article class="task-card">
  <header>
    <h3>Bài tập Flexbox</h3>
    <span class="badge">Cần làm</span>
  </header>
  <p>Đọc lý thuyết, tự code lại ví dụ và so sánh với bài giải.</p>
  <button>Đánh dấu xong</button>
</article>

CSS

.task-card {
  min-height: 240px;
  display: flex;
  flex-direction: column;
  gap: 12px;
}

.task-card header {
  display: flex;
  align-items: center;
  gap: 12px;
}

.task-card .badge {
  margin-left: auto;
}

.task-card button {
  margin-top: auto;
  align-self: flex-start;
}

Bài tập thực hành thực tế

Case study: Product card có nút mua luôn ở cuối

Dựng card sản phẩm có ảnh, tiêu đề, mô tả, giá và nút mua. Dù mô tả dài/ngắn thì nút vẫn nằm cuối card.

  1. Tạo lưới card bằng Flexbox với 3 card.
  2. Mỗi card là một flex column (nested flex).
  3. Dùng margin-top: auto để ghim nút mua ở cuối card.
  4. Dùng order để đưa badge khuyến mãi lên trước tiêu đề ở mobile.

Tiêu chí chấm

  • Các card có nút mua thẳng hàng theo đáy.
  • Badge thay đổi vị trí bằng order nhưng HTML vẫn dễ đọc.
  • Không dùng absolute để ghim nút.

Bài giải tham khảo

Lời giải dưới đây dùng nested flex + auto margin cho tình huống card thực tế.

HTML

<article class="product-card">
  <img src="https://images.unsplash.com/photo-1521572163474-6864f9cf17ab?auto=format&fit=crop&w=800&q=80" alt="Áo khoác bomber">
  <span class="badge-sale">-20%</span>
  <h3>Áo khoác bomber</h3>
  <p>Form regular, chất liệu nhẹ, phù hợp đi học và đi làm.</p>
  <strong>499.000đ</strong>
  <button>Thêm vào giỏ</button>
</article>

CSS

.product-card {
  display: flex;
  flex-direction: column;
  gap: 10px;
  min-height: 420px;
  padding: 14px;
  border-radius: 12px;
  background: #fff;
}

.product-card img {
  width: 100%;
  aspect-ratio: 4 / 3;
  object-fit: cover;
  border-radius: 10px;
}

.product-card button {
  margin-top: auto;
  align-self: flex-start;
}

@media (max-width: 640px) {
  .product-card .badge-sale { order: -1; }
}

Lỗi thường gặp

  • Dùng order thay cho HTML đúng thứ tự.
  • Quên nested flex cho header.
  • Đẩy nút bằng margin cố định làm vỡ khi nội dung dài.
