Bài 06: Holy grail layout đơn giản

Bài 06 CSS Flexbox: holy grail layout đơn giản. Có lý thuyết, ví dụ minh họa, bài tập thực hành và bài giải chi tiết.

Mục tiêu Lý thuyết Minh họa Code ví dụ Bài tập Bài giải

Mục tiêu bài học

Tạo layout sidebar cố định và content co giãn. Trọng tâm áp dụng trong bài này là Holy grail layout đơn giản.

  • Biết dùng flex: 0 0 240px cho sidebar và flex: 1 cho main.
  • Chuyển sidebar lên trên hoặc xuống dưới nội dung trên mobile.

Lý thuyết cho người mới

Layout sidebar/content là pattern cốt lõi của dashboard. Sidebar cần kích thước ổn định; content cần chiếm toàn bộ phần còn lại.

Đặt min-width: 0 cho content để bảng, text dài hoặc card bên trong không làm layout tràn ngang.

  1. Khoanh vùng phần layout liên quan đến Holy grail layout đơn giản.
  2. Chọn phần tử cha làm flex container và kiểm tra các item con trực tiếp.
  3. Viết CSS tối thiểu trước, sau đó mới thêm màu, padding, border hoặc hiệu ứng.
  4. Thu nhỏ màn hình để kiểm tra item có tràn, chồng chữ hoặc mất khoảng cách không.

.mini-demo {

Code ví dụ

Ví dụ này dùng một tình huống gần với giao diện thật để bạn thấy Holy grail layout đơn giản được áp dụng trong thực tế, không chỉ là các ô vuông mẫu.

HTML

<header class="site-header">
  <a class="logo" href="#">NenTang</a>
  <nav class="site-nav">
    <a href="#">Khóa học</a>
    <a href="#">Bài tập</a>
    <a href="#">Liên hệ</a>
  </nav>
  <button class="login-button">Đăng nhập</button>
</header>

CSS

.site-header {
  display: flex;
  align-items: center;
  gap: 18px;
  flex-wrap: wrap;
}

.site-nav {
  display: flex;
  gap: 12px;
  margin-left: auto;
}

@media (max-width: 640px) {
  .site-nav {
    width: 100%;
    margin-left: 0;
    flex-wrap: wrap;
  }
}

Bài tập thực hành

Yêu cầu

Tạo layout dashboard gồm sidebar, content header và đoạn nội dung. Sidebar cố định trên desktop, nằm trên content ở mobile. Chủ đề cần thể hiện rõ trong sản phẩm cuối: Holy grail layout đơn giản.

  1. Tạo .app-layout chứa .sidebar và .content.
  2. Đặt layout display: flex.
  3. Sidebar dùng flex: 0 0 220px, content dùng flex: 1.
  4. Thêm min-width: 0 cho content.
  5. Media query dưới 768px đổi layout sang column.

Tiêu chí đạt

  • Layout chính dùng Flexbox, không dùng table hoặc float để giả lập.
  • Khoảng cách, căn chỉnh và thứ tự item đúng với mục tiêu bài học.
  • Không có nội dung bị tràn ngang hoặc nút bị chồng lên chữ trên màn hình nhỏ.

Bài giải tham khảo

Bài giải tham khảo ưu tiên CSS ngắn và dễ đọc. Sau khi xem lời giải, hãy tự chỉ ra dòng nào điều khiển container và dòng nào điều khiển item.

<!DOCTYPE html>
<html lang="vi">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Holy grail layout đơn giản</title>
  <style>
    .site-header {
      display: flex;
      align-items: center;
      gap: 18px;
      flex-wrap: wrap;
    }
    
    .site-nav {
      display: flex;
      gap: 12px;
      margin-left: auto;
    }
    
    @media (max-width: 640px) {
      .site-nav {
        width: 100%;
        margin-left: 0;
        flex-wrap: wrap;
      }
    }
  </style>
</head>
<body>
  <header class="site-header">
    <a class="logo" href="#">NenTang</a>
    <nav class="site-nav">
      <a href="#">Khóa học</a>
      <a href="#">Bài tập</a>
      <a href="#">Liên hệ</a>
    </nav>
    <button class="login-button">Đăng nhập</button>
  </header>
</body>
</html>
