Bài 02: Named grid lines

Đặt tên cho các đường grid để placement trở nên rõ ràng và dễ bảo trì hơn.

Mục tiêu Lý thuyết Minh họa Code ví dụ Thực hành Bài giải

Mục tiêu bài học

  • Đặt tên cho grid lines bằng cú pháp [tên-line] trong grid-template-columns/rows.
  • Tham chiếu named lines khi đặt items thay vì dùng số.
  • Hiểu cách grid-template-areas tự tạo named lines ngầm định.

Lý thuyết thực tế

Named grid lines được khai báo trong dấu ngoặc vuông bên trong grid-template-columns hoặc grid-template-rows. Ví dụ: grid-template-columns: [sidebar-start] 220px [sidebar-end content-start] 1fr [content-end]. Một line có thể có nhiều tên (cách nhau bằng dấu cách trong cùng một cặp ngoặc vuông).

Khi bạn đặt tên vùng trong grid-template-areas, CSS tự động tạo ra named lines với format tên-vùng-start và tên-vùng-end. Đây là lý do đôi khi bạn thấy selector như grid-column: sidebar-start / sidebar-end hoạt động mà không cần khai báo thêm.

  1. Đặt tên cho lines trong grid-template-columns bằng cú pháp [tên].
  2. Một line có thể có nhiều tên: [sidebar-end content-start].
  3. Dùng tên line thay số khi đặt item: grid-column: sidebar-start / content-end.
  4. Code dễ đọc hơn và dễ refactor khi thay đổi số cột.

Minh họa trực quan

Layout với named lines rõ ràng cho sidebar và content area.

Header (sb-start / ct-end)
Sidebar (sb-start / sb-end)
Content (ct-start / ct-end)

Code tạo minh họa

.grid {
  grid-template-columns:
    [sb-start] 180px [sb-end ct-start] 1fr [ct-end];
}
.header  { grid-column: sb-start / ct-end; }
.sidebar { grid-column: sb-start / sb-end; }
.content { grid-column: ct-start / ct-end; }

Code ví dụ

Layout dùng named lines thay vì số để dễ bảo trì.

CSS với số (khó bảo trì)

.layout {
  grid-template-columns: 220px 1fr 200px;
}
.main { grid-column: 2 / 3; }
/* Nếu thêm cột, phải sửa số */

CSS với named lines (dễ bảo trì)

.layout {
  grid-template-columns:
    [nav-start] 220px [nav-end main-start]
    1fr [main-end aside-start]
    200px [aside-end];
}
.main { grid-column: main-start / main-end; }
/* Tên rõ ràng, không cần sửa khi thêm cột */

Yêu cầu đề bài

Tạo grid với named lines cho header-area, content-area và footer-area theo chiều dọc. Khác với ví dụ — named lines cho rows (chiều dọc), không phải columns.

  1. Định nghĩa grid-template-rows: [hd-s] 60px [hd-e ct-s] 1fr [ct-e ft-s] 50px [ft-e].
  2. Tạo 3 div và đặt chúng vào đúng vùng bằng named lines.
  3. Header: grid-row: hd-s / hd-e.
  4. Content và Footer tương tự.

Tiêu chí chấm

  • Header, content, footer xếp theo chiều dọc đúng thứ tự.
  • Dùng named lines (không dùng số) để đặt items.
  • Container có min-height: 400px.

Bài giải tham khảo

<!DOCTYPE html>
<html lang="vi">
<head>
  <meta charset="UTF-8">
  <style>
    * { box-sizing: border-box; margin: 0; }
    body { font-family: sans-serif; padding: 20px; background: #f4f7fb; }
    .page {
      display: grid;
      grid-template-columns: 1fr;
      grid-template-rows:
        [hd-s] 60px [hd-e ct-s] 1fr [ct-e ft-s] 50px [ft-e];
      gap: 12px;
      min-height: 400px;
    }
    .page-header {
      grid-row: hd-s / hd-e;
      background: #e85d4f;
      color: white;
      padding: 16px;
      border-radius: 8px;
      font-weight: 700;
      display: grid;
      place-items: center;
    }
    .page-content {
      grid-row: ct-s / ct-e;
      background: white;
      padding: 16px;
      border-radius: 8px;
      border: 1px solid #e2e8f0;
    }
    .page-footer {
      grid-row: ft-s / ft-e;
      background: #1e1b4b;
      color: white;
      padding: 16px;
      border-radius: 8px;
      font-weight: 700;
      display: grid;
      place-items: center;
    }
  </style>
</head>
<body>
  <div class="page">
    <div class="page-header">Header (hd-s / hd-e)</div>
    <div class="page-content">Content (ct-s / ct-e)</div>
    <div class="page-footer">Footer (ft-s / ft-e)</div>
  </div>
</body>
</html>
