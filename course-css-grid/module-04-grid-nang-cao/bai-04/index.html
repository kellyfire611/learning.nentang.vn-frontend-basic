Bài 04: CSS Grid và Accessibility

Đảm bảo Grid layout không phá vỡ accessibility — thứ tự DOM, keyboard navigation, và screen readers.

Mục tiêu Lý thuyết Minh họa Code ví dụ Thực hành Bài giải

Mục tiêu bài học

  • Hiểu tại sao CSS Grid có thể gây vấn đề accessibility khi thay đổi visual order.
  • Dùng semantic HTML + Grid để layout đẹp mà vẫn accessible.
  • Tránh các anti-patterns: dùng order property và dense packing cho content quan trọng.

Lý thuyết thực tế

DOM order = accessibility order. Screen readers và keyboard navigation (Tab key) theo DOM order, không theo visual order. CSS Grid có thể thay đổi visual order mà không thay đổi DOM order — điều này rất tiện cho layout nhưng nguy hiểm cho accessibility. Ví dụ: order property, negative grid coordinates, hay grid-auto-flow: dense đều thay đổi visual order.

Rules: (1) DOM order phải có nghĩa logic khi đọc từ trên xuống, không có CSS. (2) Chỉ dùng visual reordering cho nội dung thuần visual (không thể focus/interact). (3) Navigation menus: DOM order phải giống visual order để keyboard nav hợp lý. (4) Dùng semantic HTML elements: nav, main, aside, header, footer — screen reader nhận diện được landmarks này.

  1. Viết HTML theo thứ tự đọc logic, sau đó mới dùng Grid để reorder visually.
  2. Kiểm tra accessibility bằng Tab key — focus order phải hợp lý.
  3. Test với screen reader hoặc dùng Chrome DevTools > Accessibility.
  4. Thêm aria-label cho landmark regions nếu cần.

Minh họa trực quan

So sánh DOM order vs visual order.

✅ DOM order (screen reader đọc):

1. Logo
2. Navigation
3. Main content
4. Sidebar

⚠️ Visual order (dùng order property):

2. Navigation (moved)
3. Main content
1. Logo (moved)
4. Sidebar

DOM order ≠ Visual order → Tab order sai!

Semantic HTML + Grid

<!-- Dùng semantic elements -->
<header>...</header>
<nav>...</nav>
<main>...</main>
<aside aria-label="Related content">...</aside>
<footer>...</footer>

Code ví dụ

Accessible page layout với semantic HTML và Grid.

HTML (Accessible)

<!-- DOM order = reading order -->
<body class="page">
  <header role="banner">
    <a href="#main" class="skip-link">
      Bỏ qua navigation
    </a>
    <h1>Site Name</h1>
  </header>
  <nav aria-label="Main navigation">
    <a href="/">Home</a>
    <a href="/about">About</a>
  </nav>
  <main id="main">
    <h2>Page content</h2>
    <p>...</p>
  </main>
  <aside aria-label="Sidebar">
    ...
  </aside>
  <footer>...</footer>
</body>

CSS

.page {
  display: grid;
  grid-template-areas:
    "header  header"
    "nav     nav"
    "main    aside"
    "footer  footer";
}
/* Skip link cho keyboard users */
.skip-link {
  position: absolute;
  left: -9999px;
  top: 0;
}
.skip-link:focus {
  left: 0;
  z-index: 999;
  background: #7c3aed;
  color: white;
  padding: 8px 16px;
}
/* Không dùng 'order' property
   để thay đổi thứ tự focused elements */

Yêu cầu đề bài

Tạo accessible card grid với focus styles rõ ràng và semantic markup. Mỗi card phải có thể focus bằng Tab và có visible focus ring.

  1. Tạo grid 3 cột với 6 article cards.
  2. Mỗi card dùng <article> với <h3> và <a> link.
  3. CSS: a:focus { outline: 3px solid #7c3aed; outline-offset: 2px }.
  4. DOM order = đọc từ trái qua phải, trên xuống dưới.

Tiêu chí chấm

  • Cards có focus ring khi Tab qua.
  • Semantic HTML đúng.
  • DOM order hợp lý.
