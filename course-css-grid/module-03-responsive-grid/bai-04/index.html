Bài 04: Sidebar + content layout

Xây dựng layout sidebar + main content — pattern phổ biến nhất cho dashboards và web apps.

Mục tiêu Lý thuyết Minh họa Code ví dụ Thực hành Bài giải

Mục tiêu bài học

  • Tạo sidebar cố định + main content co giãn với grid-template-columns: 260px 1fr.
  • Sidebar sticky với position: sticky; top: 0; height: 100vh; overflow-y: auto.
  • Responsive: sidebar collapse thành drawer hoặc ẩn trên mobile.

Lý thuyết thực tế

Sidebar layout là pattern quan trọng nhất cho admin dashboards và web apps. Cấu trúc cơ bản: grid 2 cột với cột 1 cố định (sidebar) và cột 2 co giãn (main). Sử dụng grid-template-columns: 260px 1fr hoặc minmax(200px, 260px) 1fr để sidebar có thể co nhỏ trên màn hình nhỏ hơn.

Sidebar sticky: đặt position: sticky; top: 0; height: 100vh; overflow-y: auto lên sidebar để nó cố định khi scroll main content. Main content thường có overflow-y: auto; height: 100vh hoặc đơn giản là để trang scroll bình thường. Trên mobile, dùng media query đổi grid thành 1 cột và ẩn sidebar, hiển thị hamburger menu.

  1. Đặt body/page: display: grid; grid-template-columns: 260px 1fr; min-height: 100vh.
  2. Sidebar: position: sticky; top: 0; height: 100vh; overflow-y: auto.
  3. Main content: scroll bình thường hoặc có header cố định bên trong.
  4. Media query <768px: grid-template-columns: 1fr và sidebar { display: none }.

Code ví dụ

Admin dashboard layout hoàn chỉnh.

HTML

<div class="app">
  <nav class="sidebar">
    <div class="logo">Admin</div>
    <a href="#">Dashboard</a>
    <a href="#">Sản phẩm</a>
    <a href="#">Đơn hàng</a>
    <a href="#">Cài đặt</a>
  </nav>
  <main class="content">
    <h1>Dashboard</h1>
    ...
  </main>
</div>

CSS

.app {
  display: grid;
  grid-template-columns: 260px 1fr;
  min-height: 100vh;
}
.sidebar {
  background: #1e1b4b;
  position: sticky;
  top: 0;
  height: 100vh;
  overflow-y: auto;
  padding: 24px 16px;
}
.content { padding: 32px; }
@media (max-width: 768px) {
  .app { grid-template-columns: 1fr; }
  .sidebar { display: none; }
}

Yêu cầu đề bài

Tạo sidebar layout cho trang docs/documentation với table of contents ở sidebar. Khác với ví dụ — không phải admin dashboard mà là trang documentation.

  1. Sidebar bên trái (200px): "Mục lục" với các heading links (4-5 items).
  2. Main content: bài viết giả với h1, h2, đoạn văn (không cần nội dung thật).
  3. Sidebar và main đều có padding hợp lý.
  4. Màu sắc: sidebar nền nhạt, main nền trắng.

Tiêu chí chấm

  • Sidebar 200px, main co giãn.
  • Layout docs trông chuyên nghiệp.
  • Links trong sidebar hover có màu.
