Bài 05: Responsive với media queries

Kết hợp CSS Grid với media queries để kiểm soát layout chính xác theo breakpoints.

Mục tiêu Lý thuyết Minh họa Code ví dụ Thực hành Bài giải

Mục tiêu bài học

  • Thay đổi grid-template-columns và grid-template-areas theo breakpoints.
  • Mobile-first approach: bắt đầu với 1 cột, thêm cột ở màn hình lớn hơn.
  • Biết khi nào dùng auto-fit (không cần query) và khi nào cần media query cho kiểm soát chính xác hơn.

Lý thuyết thực tế

Mobile-first với Grid: Bắt đầu với layout 1 cột (mobile), dùng @media (min-width: 768px) để thêm cột tablet, @media (min-width: 1024px) cho desktop. Đây là approach chuẩn — CSS nhỏ nhất cho mobile, CSS thêm vào cho màn hình lớn.

Với Grid, thay đổi grid-template-columns trong media query cũng tự động điều chỉnh vị trí items nếu bạn không dùng explicit placement. Với grid-template-areas, thay đổi cả areas để cấu trúc layout thay đổi hoàn toàn giữa mobile và desktop — rất linh hoạt. Tên areas phải nhất quán giữa các breakpoints nếu bạn dùng cùng elements.

  1. Mặc định: grid-template-columns: 1fr (1 cột cho mobile).
  2. @media (min-width: 640px): 2 cột.
  3. @media (min-width: 1024px): 3 hoặc 4 cột.
  4. Nếu dùng areas: định nghĩa lại areas cho mỗi breakpoint.

Mobile-first pattern

/* Mobile: 1 cột */
.grid { grid-template-columns: 1fr; }
/* Tablet: 2 cột */
@media (min-width: 640px) {
  .grid { grid-template-columns: 1fr 1fr; }
}
/* Desktop: 3 cột */
@media (min-width: 1024px) {
  .grid { grid-template-columns: repeat(3, 1fr); }
}

Code ví dụ

Layout thay đổi hoàn toàn giữa mobile và desktop với template areas.

HTML

<div class="page">
  <header>Header</header>
  <nav>Navigation</nav>
  <main>Main Content</main>
  <aside>Sidebar</aside>
  <footer>Footer</footer>
</div>

CSS

/* Mobile: 1 cột, stacked */
.page {
  display: grid;
  grid-template-areas:
    "header" "nav"
    "main"   "sidebar"
    "footer";
}
/* Desktop: multi-column */
@media (min-width: 1024px) {
  .page {
    grid-template-columns: 200px 1fr 200px;
    grid-template-areas:
      "header  header  header"
      "nav     main    sidebar"
      "footer  footer  footer";
  }
}

Yêu cầu đề bài

Tạo pricing section responsive với 3 pricing tiers — 1 cột trên mobile, 3 cột trên desktop. Khác với ví dụ — không dùng template-areas, dùng grid-template-columns thay đổi theo breakpoint.

  1. Mobile (mặc định): grid-template-columns: 1fr, cards stack dọc.
  2. Tablet (≥640px): grid-template-columns: 1fr 1fr.
  3. Desktop (≥1024px): grid-template-columns: repeat(3, 1fr).
  4. Card giữa (Pro) có style nổi bật hơn (màu đậm, scale lớn hơn).

Tiêu chí chấm

  • Layout thay đổi đúng theo breakpoints.
  • 3 pricing cards với tên gói, giá, và 3-4 features.
  • Card giữa nổi bật hơn.
